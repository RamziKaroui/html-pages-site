Simple Standalone/Single Process Architecture

Architecture Key Elements Federated & Shared Memory Standalone Mode Network Scheduler Traffic Confinement

This deployment allows DDS applications and Opensplice DDS middleware libraries to be linked together into a single process.

There is no need to pre-configure a shared memory segment, as dynamic heap memory is used by the process. With zero configuration, there is no need to start the federation services because all functionality is bundled within the application process.

This deployment is useful when there is one DDS-aware application per machine and nothing to federate. It can be particularly useful for embedded platforms.
