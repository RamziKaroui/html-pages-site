Architecture Key Elements Federated & Shared Memory Standalone Mode Network Scheduler Traffic Confinement

DDS global data spaces and the applications that produce them can be organized in logical groups known as DDS partitions.

Non-Opensplice DDS implementations often use the same multicast address to spread data everywhere and rely on the DDSI protocol stack to discard unwanted data on a given node.

Opensplice DDS Network Scheduler allows different multicast and unicast addresses to be associated with the same DDS logical partition to physically confine traffic to the subnetwork where there is real interest in the produced data.
