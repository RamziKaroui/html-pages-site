Federated Architecture and Shared Memory

To rationalize memory resources when several DDS-aware applications are running on the same node, Opensplice DDS can support shared memory to minimize the need to keep data in each application's address space.

The shared memory segment is common to all local DDS applications. It can be seen as an in-memory real-time database that can be queried using SQL.

When configuring Opensplice DDS for Federated deployment, data is physically stored only once on a machine federation. Smart administration still provides each subscriber with its own private view on the data space.
